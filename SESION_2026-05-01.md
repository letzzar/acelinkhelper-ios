# Sesión 2026-05-01 — Port iOS de AcelinkHelper

## Contexto

Port de `acelinkhelper-tv` (Kotlin/Android) a iOS nativo con Swift + SwiftUI.

## Decisión de arquitectura

Se evaluaron tres opciones:

| Opción | Descripción | Decisión |
|---|---|---|
| A — KMP + SwiftUI | Kotlin compila a framework, UI en SwiftUI | Descartada — lógica compartible son <30 líneas |
| **B — Swift nativo + SwiftUI** | Reescritura completa en Swift | **Elegida** |
| C — Flutter | Un codebase para Android e iOS | Descartada — overhead excesivo, UI no nativa |

**Razón:** La "lógica de negocio" del proyecto original es trivial (transformar una URL, guardar una IP, abrir VLC). KMP añade complejidad de build sin beneficio real.

---

## Lo que se construyó

### Target principal: `AcelinkHelper`

#### `StreamHandler.swift`
Transforma `acestream://ID` → `http://NAS_IP:6878/ace/getstream?id=ID`.
Maneja dos formatos: ID como host (`acestream://ID`) e ID como path (`acestream:///ID`).

#### `VLCLauncher.swift`
Abre VLC usando el esquema `vlc://` + URL del stream. Verifica con `canOpenURL` si VLC está instalado antes de intentarlo. Errores tipados (`LaunchError`).

#### `Info.plist`
- Registra `acestream://` como URL scheme propio (`CFBundleURLTypes`)
- Declara `vlc` en `LSApplicationQueriesSchemes` (requerido por iOS 9+ para `canOpenURL`)

#### `AppState.swift`
- `ObservableObject` que centraliza estado UI y lógica
- Persiste NAS IP en **App Group UserDefaults** (`group.com.acelinkhelper.ios`) para compartirla con la extensión
- `handleIncomingURL` valida, transforma y lanza VLC con manejo de errores
- `deviceIP` detecta IP local vía POSIX `getifaddrs` en interfaz `en0`

#### `ContentView.swift`
- SwiftUI con tema oscuro idéntico al Android (#0d0d1a fondo, #6b8cff acento)
- QR generado con `CoreImage` nativo (`CIQRCodeGenerator`) — sin dependencias externas
- QR apunta a `http://[IP_DISPOSITIVO]:8765` (listo para el servidor HTTP)
- Input NAS IP + botón Guardar + mensaje de estado

#### `AcelinkHelperApp.swift`
- Entry point `@main`
- `onOpenURL` captura enlaces `acestream://` del sistema → delega a `AppState`

---

### Target extensión: `AcelinkHelperShare`

Motivación: el usuario preguntó sobre AirDrop. AirDrop entre iPhone/Mac ya funciona sin código extra (el `onOpenURL` lo captura). Lo que sí añade valor es una **Share Extension** — aparece en el share sheet de Safari y cualquier app.

#### `ShareViewController.swift`
- Lee URL compartida (tipo `public.url` o fallback `public.plain-text`)
- Lee NAS IP del App Group UserDefaults compartido
- Transforma URL con `StreamHandler`
- Abre VLC directo vía `extensionContext?.open(vlcURL)`
- Si VLC no está instalado → alerta con mensaje claro

#### `AcelinkHelperShare/Info.plist`
- `NSExtensionActivationRule` con predicado SUBQUERY — aparece solo para URLs y texto plano
- Bundle ID: `com.acelinkhelper.ios.share`

## Flujos implementados

```
# Flujo 1: Link directo (acestream:// desde browser u otra app)
Sistema detecta acestream:// → AcelinkHelperApp.onOpenURL
  → AppState.handleIncomingURL
    → StreamHandler.buildStreamURL
      → VLCLauncher.launch(vlc://stream_url)

# Flujo 2: Share Extension (desde share sheet de cualquier app)
Usuario pulsa Compartir → AcelinkHelper Share
  → ShareViewController.extractAndProcess
    → Lee NAS IP del App Group
      → StreamHandler.buildStreamURL
        → extensionContext.open(vlc://stream_url)
```
